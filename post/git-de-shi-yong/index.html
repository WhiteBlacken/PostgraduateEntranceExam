Git的使用

2021-05-06

本地使用

初始化git仓库

  • git init

查看

  • git status

操作暂存区

  • git add xxx.xxx
  • git add -u
    • 只添加被跟踪的文件
  • git rm --cache xxx.xxx

提交（只操作暂存区的文件）

  • git commit -m “first commit”
  • git commit（常用）

工作区向版本库对齐

  • git reset --hard

远端仓库

设置

  • git remote add origin 地址

查看

  • git remote -v

本地提交到github

  • git push origin -u origin main

浏览工作区状态与历史

  • git status -u
    • no -只显示已经跟踪的文件
    • normal -显示没有被跟踪的文件
    • all -未跟踪的文件夹内的文件

新建文件夹

  • mkdir newfolder

操作文件

  • 基本信息
    • git config --global user.name “your name”
    • git config --global user.email “your email”
    • --global vs --local（当前工程）
  • git add
    • filename
    • .
    • -u
      • 修改（即已托管）
    • -i
      • 交互式（其他为批处理）
  • git commit
    • 空
    • -a
      • add+commit（被跟踪的文件的才会执行）
    • -m msg

当前工作区搜索

  • git grep -n content
    • 工作区中查找
  • git grep --untracked -n content

历史搜索中查找

  • git log -G
    • 从历史修改中查找
  • git log -S
    • 显示命中次数

删除或移动文件

  • git rm
  • git mv

分支

  • 查看
    • git branch
    • git branch -avv
  • 新建分支
    • git branch branchname
    • git checkout branchname
    • git checkout -b branchname
      • 新建+切换
  • 推送分支
    • git push origin branch：branch
    • git push origin：branch
      • 删除方法
    • 两种方法

修改

  • 修改历史提交
    • git revert commitId
  • 修改上一个提交
    • git commit --amend
    • git commit --amend --author=”Author name“
  • 修改前几次提交
    • git rebase -i （交互式）

使用

  • 寻找写错代码的人
    • git blame filename
  • 找回丢失的提交
    • git reflog
    • git checkout hashcode
  • git瘦身
    • git gc
